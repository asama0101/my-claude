# PARA タスクマネージャー

PARAメソッドに基づき、タスクと情報を `/home/asama/notebook/claude-pm/` のMarkdownファイルで管理する。

---

## PARAとは

| カテゴリ | 意味 | 特徴 |
|---|---|---|
| **Projects** | 期限のある具体的なゴール | Todoあり・進捗管理・完了したらArchiveへ |
| **Areas** | 継続的に維持する責任・習慣 | 基準を保つもの・終わりがない |
| **Resources** | 関心・参照情報のストック | いつか役立つメモ・リンク・学習ノート |
| **Archives** | 完了・非アクティブになったP/A/R | 参照用・再活性化可能 |

---

## データ構造

```
/home/asama/notebook/claude-pm/
├── index.md          # PARAインデックス（全体の地図）
├── projects/         # P: 期限のある短期ゴール
├── areas/            # A: 継続的な責任・習慣
├── resources/        # R: 参照・関心情報
├── archives/         # Arch: 完了・非アクティブ
└── daily/            # 日次レビュー記録
```

### index.md フォーマット

```markdown
# PARA インデックス

最終更新: YYYY-MM-DD

## 📁 Projects（プロジェクト）
| 名前 | ファイル | 進捗 | 期限 |
|---|---|---|---|
| 大掃除 | projects/oosouji.md | 0/3 (0%) | - |

## 🏔️ Areas（領域）
| 名前 | ファイル | 状態 |
|---|---|---|

## 📚 Resources（リソース）
| 名前 | ファイル |
|---|---|

## 🗄️ Archives（アーカイブ）
| 名前 | 元カテゴリ | アーカイブ日 |
|---|---|---|
```

### Todoのステータス

Todoはステータス別の見出し（`###`）でグループ化する。インラインにステータスタグは付けない。

| Status | 意味 | チェックボックス |
|---|---|---|
| **Inbox** | 捕捉済みだがまだ整理・着手未定 | `[ ]` |
| **Open** | アクティブに進行中（デフォルト） | `[ ]` |
| **Pending** | 何かを待っている（他者・イベント・情報） | `[ ]` |
| **Review** | 確認・レビュー中（完了前の最終チェック） | `[ ]` |
| **Close** | 完了 | `[x]` |
| **Cancel** | キャンセル | `[ ]` |

新規TodoはデフォルトでOpenセクションに追加する。Pendingには「待ち: 〇〇」を付けると管理しやすい。空のステータスセクションは省略する。

### Projectファイル フォーマット

```markdown
# プロジェクト名

**PARA**: Project | **作成**: YYYY-MM-DD | **状態**: 進行中

説明文（任意）

---

## Todo

### Open
- [ ] 🔴 タスク名 | 期限: 2026-05-05
- [ ] 🟡 別のタスク

### Pending
- [ ] 🟡 上司の承認待ち | 待ち: 上司の承認

### Close
- [x] 🔴 完了タスク | 完了: 2026-05-01

---

## 進捗

完了: 1 / 3 (33%)
```

進捗の分母はCancel・Inboxを除いたタスク数、分子はCloseの数とする。

## 優先度

| 記号 | 意味 |
|---|---|
| 🔴 | 高：緊急・重要 |
| 🟡 | 中：通常（デフォルト） |
| 🟢 | 低：余裕があれば |

## ファイル命名

スラッグは日本語をローマ字または短い英語で表現し、ハイフン区切り。
- 例：「大掃除」→ `projects/oosouji.md`
- 例：「健康管理」→ `areas/health.md`
- 例：「Python学習メモ」→ `resources/python.md`

## 操作手順

### 共通：事前確認

操作前に `index.md` を Read して現在の状態を把握する。存在しなければ上記フォーマットで初期化する。

### 登録（P/A/R 共通）

1. 名前・PARAカテゴリ・説明（任意）を確認する
2. スラッグを決定し、`{category}/{slug}.md` を Write で作成する
3. `index.md` の該当セクションに追加する
4. 「登録しました」と伝えるだけでよい。余計な確認質問はしない

### Todo追加

1. 対象ファイルを Read する
2. 優先度・ステータス・期限が未指定なら🟡中・Open・期限なしをデフォルトで適用（確認不要）
3. 該当ステータスの `###` セクションの末尾に追加する。セクションがなければ作成する
4. Pendingなら「待ち: 〇〇」を末尾に付ける
5. `## 進捗` を更新（Cancel・Inbox以外の分母+1）し、`index.md` も更新する

### ステータス変更

「〇〇をPendingにして」「〇〇をReviewに変えて」など：
1. 対象ファイルを Read する
2. 該当行を旧ステータスセクションから切り取り、新ステータスセクションへ移動する
3. セクションが空になったら見出しごと削除する

### Todo完了（Closeにする）

1. 対象ファイルを Read する
2. 該当行を `### Open`（または現在のセクション）から `### Close` セクションへ移動する
3. チェックボックスを `[x]` にし、末尾に `| 完了: YYYY-MM-DD` を追加する
4. `## 進捗` を更新（Close数+1）し、`index.md` も更新する

### Todo削除

確認なしで削除する。ユーザーの判断を尊重する。

### アーカイブ（完了したProjectなど）

1. 対象ファイルを Read して内容を確認する
2. ファイルを `archives/{slug}.md` にコピーし、`**PARA**: Archive | アーカイブ日: YYYY-MM-DD` に更新する
3. 元ファイルを削除し（ファイル削除はユーザー確認を取る）、`index.md` を更新する

### 一覧表示

`index.md` を Read してP/A/R/Arch別に整形して表示する。

## 今日のタスク表示

「今日のタスクは？」「今日やることは？」で起動。

1. `index.md` から全ファイルを取得し、Read する
2. 未完了Todoをフィルタ：**期限超過** / **今日が期限** / **14日以内** / **🔴期限なし**
3. 以下のフォーマットで表示：

```markdown
## 📋 今日のタスク（YYYY-MM-DD）

### ⚠️ 期限超過
- 🔴 [大掃除] クローゼットの片付け | 期限: 2026-05-02（N日超過）

### 📅 今日が期限
（該当なし）

### 🗓️ 2週間以内
- 🟡 [英語学習] リスニング練習 | 期限: 2026-05-12（あとN日）

### 🔴 高優先度（期限なし）
- 🔴 [読書] 『ファクトフルネス』読了
```

## 日次レビュー

「振り返り」「1日終わり」「今日を振り返って」「日次レビュー」で起動。

1. `index.md` から全Projectファイルを Read する
2. 今日完了したタスク（`完了: YYYY-MM-DD` が今日）を収集する
3. 期限超過・高優先度未完了を収集する
4. 以下のフォーマットを表示し、振り返りコメントをユーザーに促す：

```markdown
## 🌙 日次レビュー（YYYY-MM-DD）

### ✅ 今日完了したこと
- 🔴 [大掃除] クローゼットの片付け

### 📋 未完了・繰り越し
- 🟡 [大掃除] ベッド足元に検証環境を設置

### 💭 振り返り
> （ここにコメントをどうぞ）

### 🎯 明日の優先タスク
（未完了から自動提案：期限が近いもの・高優先度のもの）
```

5. ユーザーがコメントを入力したら、`daily/YYYY-MM-DD.md` に保存する

## 全体進捗表示

「進捗を見せて」「全体を確認したい」で起動。

```markdown
## 📊 全体進捗

### 📁 Projects
| プロジェクト | 進捗 | バー |
|---|---|---|
| 大掃除 | 1/3 (33%) | ███░░░░░░░ |

### 🏔️ Areas  Active: N件

### 📚 Resources  登録: N件
```

進捗バーは10文字（`█`完了・`░`未完了）。

---

## 注意事項

- `index.md` とファイルの進捗は常に同期する
- 日付は `YYYY-MM-DD` 形式に統一。「5/10」→ 年を補完
- 名前があいまいで複数一致するときだけ確認する。それ以外は推測して進める
- ファイル丸ごと削除するときだけユーザー確認を取る。Todo行の削除は確認不要
