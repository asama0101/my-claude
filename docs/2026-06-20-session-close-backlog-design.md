# セッションクローズ改善フロー：薄れない改善バックログ

- 日付: 2026-06-20
- 対象スキル: `~/.claude/skills/session-close-improve`（グローバル定義。my-claude では `claude/skills/...` にミラー）
- ステータス: 設計確定（実装計画はこの後 writing-plans で作成）

## 背景と課題

現状の `session-close-improve` は **1ショット・その場・自己評価** のフローで、各観察に対し「即時実行」か「寝かせる（ラダー① メモリ記録のみ）」の二択しかない。

寝かせた項目を **後で必ず拾い直す閉ループが無い** ため、以下が同じ穴から漏れる：

- **見逃し**：新スキル/エージェントにすべき種が「メモで寝かせ」で事実上ロスト。
- **実行漏れ**：`scripts/sync.sh` 同期や保留更新が「記録だけ」で実行されない。

ユーザーの動機（複数選択）：振り返りの質・更新の実行漏れ・フロー全体の重さ・新規作成の提案。
昇格の判断軸は **人間の承認一本**（出現回数や実害による自動昇格はしない）。
方針は **最軽量（案A）**：フックは触らず、スキル定義の編集のみ。

## 解決アイデア

**薄れない「改善バックログ」**を1枚用意する。寝かせる種・保留更新・未実行同期を構造化して溜め、クローズ時に全件そのまま提示して人間が「今やる」を選ぶ。やったら消える、やるまで出続ける＝閉ループ。

モデル側の責務は **捕捉・重複排除・提示** のみ。悩ましい昇格判断は人間に委ねるため、その場の認知負荷は増えない（むしろ二択での即断を減らす）。

## 設計

### 1. バックログ・ファイル

- **置き場所**：`~/.claude/improvement-backlog.md`（グローバル1枚）。
  - どのプロジェクトからクローズしても同じ1枚を見るため、グローバル定義の種が確実に再浮上する。
  - **sync ミラー対象外**：`scripts/sync.sh` のホワイトリストに入れない。ローカル限定の作業状態として扱う。
- **形式（最小）**：3セクションの Markdown 箇条書き。状態は「行の有無」で表現（完了＝行削除、保留＝残す）。ステータス列は持たない。

```markdown
## 新規作成候補
- [skill] runbook差分の自動要約 — 手作業で3手かかった (初出: 2026-06-20)

## 保留更新
- [agent:reviewer-test] plugin形式スキル検出漏れ — grepが~/.claude/skillsのみ (初出: 2026-06-20)

## 未実行同期
- [sync] グローバル定義編集後の scripts/sync.sh 未実行 (初出: 2026-06-20)
```

- **タグ規約**：先頭に `[type]`（`skill` / `agent:<name>` / `reference:<name>` / `sync` / `proj:<name>` 等）。プロジェクト固有項目は `[proj:xxx]` で併記。
- **重複排除キー**：同 type ＋同対象。既存があれば追記しない。任意で末尾に `／再: YYYY-MM-DD` を付け、再発の証拠だけ人間に見せる（自動昇格はしない）。

### 2. フローへの組み込み（SKILL.md の改変点）

新ステップは増やさず、既存ステップの分岐を変える。

**Step 1（振り返り）— 「寝かせる」の行き先を変更**
- ラダー①「メモリ記録のみ（寝かせる）」に落ちる `[実害]/[先回り]/新規作成候補` は、メモではなく **バックログへ1行 append（重複排除付き）**。
- 即時実行ゲートを通る項目は従来どおり Step 2 へ直行（バックログを経由しない）。

**Step 2.0（新サブステップ）— バックログ提示**
- バックログが空なら「保留なし」1行で **即スキップ**。
- 空でなければ **全件をコンパクトに列挙**し、「今やる項目」を番号でユーザーに選ばせる。未選択はそのまま残す。
- 選ばれた項目だけ type に応じて既存機構へルーティング：
  - `保留更新` → 既存ラダー②〜⑥／`claude-md-management:revise-claude-md` → `claude-md-management:claude-md-improver`
  - `新規作成候補` → ⑥新規作成（必要なら `skill-creator`）
  - `未実行同期` → `bash scripts/sync.sh`

**完了＝行削除（閉ループの要）**
- 実行・反映できた項目はバックログから該当行を削除。やり残しは残す＝次回また出る。
- グローバル定義を触ったのに sync 未実行なら、Step 3 手前で **自動的に `未実行同期` をバックログへ積む**（記録だけで終わらせない）。

### 3. 肥大・陳腐化の防止（軽さの担保）

- **空なら即スキップ**：保留が無いセッションは認知負荷ゼロ。
- **再litigateしない**：提示は列挙のみ。各項目を毎回議論し直さない。
- **陳腐化は人間が落とす**：ユーザーが不要と言えば即削除。自動プルーンはしない。
- **60日助言フラグ**：初出から60日超の項目に `(陳腐化?)` を **表示だけ** 付ける。昇格・削除には一切関与しないただの目印。

## スコープ外（YAGNI）

- SessionStart など **フックの変更**（案B以降。今回は案A）。
- 出現回数・実害による **自動昇格ロジック**（人間承認一本）。
- 自動プルーン／陳腐化の自動削除。
- バックログの sync ミラーリング・バージョン管理。

## 受け入れ基準

1. 寝かせる判断の項目が `~/.claude/improvement-backlog.md` に重複なく追記される。
2. クローズ時、空なら1行スキップ・非空なら全件提示され、選択した項目だけが実行される。
3. 実行完了した行が削除され、未実行の行は次回クローズで再提示される。
4. グローバル定義編集後に sync 未実行なら `未実行同期` が自動で積まれる。
5. 初出60日超の項目に `(陳腐化?)` が表示される。
6. フック（`session-stop.sh` 等）は一切変更されない。
