# 計画段階チェックリスト

## 目的

`superpowers:writing-plans` でプランを立案する際に、実装に必要な情報収集を完了させるためのチェックリスト。

## チェックリスト

計画ドキュメント（仕様書、既存計画、設計書）を読んだ後、実装フェーズに入る前に以下を確認:

### 1. ライブラリ・フレームワークの制約確認

- [ ] メインフレームワーク（FastAPI, polars, TimescaleDB等）の型チェック・Decimal 型など特殊な要件を context7 で確認したか
- [ ] サードパーティライブラリ（pgcopy, psycopg2等）の制約・制限事項を確認したか
- [ ] データベーススキーマに型制約（NUMERIC, INTEGER等）がないか確認したか

**例**:
- Python ETL: pgcopy の `CopyManager` が Decimal 型をサポートするか確認 → 実装後に発見してやり直した

### 2. サブエージェント（多視点レビュー）

- [ ] 複雑な設計決定があるか（アーキテクチャ、並列処理、キャッシング等）
- [ ] セキュリティに関わる要件があるか（認証、入力検証、ファイルアクセス）
- [ ] パフォーマンス要件がある場合、アーキテクチャを architect エージェントに相談したか

**例**:
- Python ETL: ProcessPoolExecutor の設定、pgcopy バイナリ COPY のパフォーマンス特性を事前検証すべきだった

### 3. テスト戦略

- [ ] TDD を適用するフェーズか（新機能・バグ修正）、既存テスト検証フェーズか区別したか
- [ ] テストカバレッジ目標（80%以上）が達成可能か確認したか
- [ ] モック・フィクスチャの計画が立っているか

### 4. 既存ドキュメント・制約との整合性

- [ ] CLAUDE.md の「Gotchas」「環境セットアップ」に記載されている制約が考慮されているか
- [ ] 過去セッションの lessons.md に既知の問題がないか確認したか

## 違反時の対応

各チェック項目をスキップした場合、**計画レビュー段階で指摘されます**。
実装開始前に必ず対応してください。
