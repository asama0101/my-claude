# 設計仕様: Claude Code プロジェクトテンプレート完成

**日付:** 2026-05-10

**対象:** Python バックエンド業務システム向け Claude Code テンプレート

## 背景と課題

`03_project-memory/` は新規プロジェクトの `.claude/` に配置するテンプレート群だったが、以下の課題があった:

1. `プロジェクトCLAUDE.md` のプレースホルダーが抽象的で、記入例がなく Claude が意図を読み取りにくい
2. テンプレートを新プロジェクトに適用する手順書がなく、摩擦が大きい
3. フェーズ移行・PR提出・リリース前の確認作業が体系化されていない

## 設計決定

### ファイル構成

```
.claude/
├── プロジェクトCLAUDE.md   ← 更新: プレースホルダーを Python バックエンド例で書き換え
├── dev.md                  ← 変更なし
├── docs.md                 ← 変更なし
├── review.md               ← 変更なし
├── setup.md                ← 新規: 新プロジェクト適用手順
└── checklist.md            ← 新規: 開発チェックリスト
```

**分割の理由:** 各ファイルの責務を単一にし、既存の `.claude/` 分割構成に自然に馴染ませる。

### プレースホルダーの書き換え方針

抽象的な `<...>` をそのまま削除するのではなく、HTML コメント形式の「記述ガイド行」＋具体例という構成にした。これにより:
- 新プロジェクトで何を書けばよいかが一目でわかる
- コピー後に不要なコメント行だけ削除すればよい
