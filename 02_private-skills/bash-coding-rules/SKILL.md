# Bash シェルスクリプト コーディングルール

bash 4.x 以降を前提（POSIX sh 互換は要求しない）。

## このスキルの使い方

| 作業種別 | 適用方法 |
|---|---|
| **新規スクリプト生成** | コアルール（C1〜C11）を必ず満たし、50行超・継続メンテ・共有予定の場合は拡張ルール（E1〜E3）も適用する |
| **既存スクリプトの編集** | 変更箇所と影響範囲でルール違反があれば同時に修正する |
| **コードレビュー** | 違反箇所を「ルール番号 + 理由 + 修正例」の形式で列挙する |

---

## 適用範囲

- **コアルール（C1〜C11）**：すべての bash スクリプトに必須
- **拡張ルール（E1〜E3）**：以下のいずれかに該当する場合に追加適用する
  - 行数が **50 行を超える**
  - 継続的にメンテナンスされる（運用ツール、CI 組込み等）
  - 複数人で共有・引き継ぎする予定がある

# コアルール（C1〜C11）

## C1. ファイル先頭テンプレート

すべてのスクリプトはこのテンプレートで始める。`set -euo pipefail` と `IFS=$'\n\t'` はバグの早期発見と予期しない挙動防止のために不可欠：

```bash
#!/usr/bin/env bash
# -----------------------------------------------------------------------------
# <スクリプトの目的を 1〜2 行で>
#
# Usage:
#   ./script_name.sh [options] <args>
#
# Example:
#   ./script_name.sh --verbose /path/to/target
# -----------------------------------------------------------------------------
set -euo pipefail
IFS=$'\n\t'
```

- shebang は `#!/usr/bin/env bash`（`/bin/bash` 直指定はしない）。
- 冒頭コメントには **目的・使い方・実行例** を明記する。

---

## C2. 命名規則

- ファイル名・関数名・ローカル変数：`snake_case`
- 環境変数・グローバル定数：`UPPER_SNAKE_CASE`
- プライベート関数：先頭 `_`

---

## C3. 変数・展開

- 変数展開は **必ずダブルクォート** で囲む：`"${var}"`, `"${array[@]}"`
- 中括弧形式 `${var}` を基本とする（曖昧さ回避）。
- コマンド置換は `$(...)` を使う（バッククォートは禁止）。
- 読み取り専用の定数は `readonly` で宣言する。

---

## C4. 条件分岐

- 条件式は `[[ ... ]]` を使う（`[ ... ]` は使わない）。
- 数値比較は `(( ... ))` を使う。

---

## C5. 関数

- スタイルは `function name() { ... }` で統一。
- 引数 `$1`, `$2` は冒頭で必ず `local` 変数に代入する（グローバル汚染を防ぐ）。
- 戻り値は **整数の終了ステータスのみ**（0-255）。文字列は `echo` で stdout に出して呼び出し側でキャプチャする。
- 関数の前に概要・引数・戻り値のコメントを書く：

```bash
# 指定ホストへの疎通確認を行う。
# Args:
#   $1 - ホスト名または IP
#   $2 - タイムアウト秒（省略時 5）
# Returns:
#   0: 成功 / 1: 失敗
function check_reachability() {
    local host="$1"
    local timeout="${2:-5}"
    ping -c 1 -W "${timeout}" "${host}" > /dev/null 2>&1
}
```

---

## C6. エラーハンドリング・クリーンアップ

- `set -euo pipefail` を前提に、意図的に失敗を許容する箇所だけ `|| true` や明示的分岐で対応する。
- 一時ファイル等のクリーンアップは `trap` で確実に行う（スクリプトが途中で死んでも残骸を残さない）：

```bash
readonly TMP_DIR=$(mktemp -d)
trap 'rm -rf "${TMP_DIR}"' EXIT
```

- エラーメッセージは **stderr** に出す（`>&2`）。
- 終了コード：`0` = 成功、`1` = 一般失敗、`2` = 引数不正。

---

## C7. ログ出力

- ログは **stderr に出す**（stdout はスクリプトの「データ出力」用に空けておく）。
- ログ関数：

```bash
function log_info()  { echo "[$(date +'%F %T')] [INFO]  $*" >&2; }
function log_error() { echo "[$(date +'%F %T')] [ERROR] $*" >&2; }
function log_debug() { [[ "${DEBUG:-0}" == "1" ]] && echo "[$(date +'%F %T')] [DEBUG] $*" >&2 || true; }
```

- DEBUG ログは `DEBUG=1` 環境変数で切り替え可能にする。
- 機密情報（パスワード、API キー、トークン）はログに出さない。

---

## C8. 引数パース

- 単純な位置引数は `$1`, `$2` をそのまま使う。
- フラグ・長いオプションが必要なら **case 文での手動パース** を使う。
- `--help` / `-h` は必ず実装し、`usage()` で使い方を表示する。

```bash
while [[ $# -gt 0 ]]; do
    case "$1" in
        -v|--verbose) VERBOSE=1; shift ;;
        -d|--dry-run) DRY_RUN=1; shift ;;
        -h|--help)    usage ;;
        --)           shift; break ;;
        -*)           log_error "Unknown option: $1"; usage ;;
        *)            break ;;
    esac
done
```

---

## C9. 冪等性・副作用

- スクリプトは原則 **冪等** に書く（同じ入力で何度実行しても同じ結果）。
- 副作用が大きいスクリプトは **`--dry-run` モード** を実装する。
- 上書きする場合は明示的な許可フラグ（`--force` 等）を要求する。

---

## C10. 安全性

- **`eval` の使用は禁止**。動的コマンド構築は配列を使う。
- ユーザー入力をコマンドに渡す際は必ずクォートする（インジェクション対策）。

---

## C11. shellcheck（必須）

- すべての bash スクリプトは **shellcheck を通す**。警告ゼロを目標とする。
- 抑制が必要な場合は理由をコメントで明記する：

```bash
# shellcheck disable=SC2086  # 意図的に分割展開している
cmd ${args}
```

- フォーマッタは **shfmt**（`-i 4 -ci`）を推奨。

# 拡張ルール（E1〜E3）
50行超 / 継続メンテ / 共有予定の場合のみ適用する。

## E1. 外部コマンド・ループ

- 使用する外部コマンド（`jq`, `curl` 等）は冒頭で `command -v` により存在確認する。
- `cd` は `cd "${dir}" || exit 1` か、サブシェル `(cd "${dir}" && ...)` で呼び出し元を汚染しない。
- ファイル一覧ループはスペース・改行を含むファイル名対策として `mapfile` または `find -print0` + `while read -d ''` を使う：

```bash
mapfile -t files < <(find . -name '*.log')
for file in "${files[@]}"; do
    process "${file}"
done
```

---

## E2. プロジェクト構成

```
project_root/
├── scripts/
│   ├── deploy.sh
│   ├── collect_stats.sh
│   └── lib/
│       └── common.sh    # ログ関数等の共通ライブラリ
└── docs/
    └── developer.md
```

- 複数スクリプトで共有する関数は `scripts/lib/common.sh` 等に切り出し `source` で読み込む：

```bash
# shellcheck source=lib/common.sh
source "$(dirname "$0")/lib/common.sh"
```

- スクリプトには実行権限を付与する（`chmod +x`）。

---

## E3. その他

- 100 行を大きく超え、複雑な文字列処理・連想配列・データ変換が中心になる場合は **Python での再実装を検討** する。
- 長いパイプは行継続（`\`）で改行し可読性を保つ。
- スクリプトが長くなったら `main()` を定義して最下部で `main "$@"` を呼ぶ：

```bash
function main() {
    parse_args "$@"
    run_main_logic
}

main "$@"
```

- 行コメントは **「なぜそうしているか」** を中心に書く。複雑なリダイレクト・正規表現には意図を必ず書く。
